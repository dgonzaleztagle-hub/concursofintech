# Casos de Uso: Beeper Financiero Pro
**Expert Role:** Lead Systems Analyst & Use Case Strategist

Este documento detalla los flujos de interacción principales que permiten al ciudadano recuperar el control de su salud financiera.

---

## 1. Auditoría Proactiva (Push Sentinel)
**Actor:** Usuario Bancarizado / Sistema Beeper
**Pre-condición:** El usuario tiene activada la "Billetera Sentinel".

### Flujo Principal:
1. El Sistema detecta un cobro sospechoso (ej. seguro duplicado) en la billetera digital del usuario.
2. El Beeper emite una **Alerta Push** inmediata en el dispositivo.
3. El Usuario pulsa la alerta, abriendo el Beeper en el LCD.
4. El Sistema realiza un escaneo automático (IA) del perfil financiero.
5. El Sistema muestra el ahorro potencial y el diagnóstico legal.

**Valor Generado:** Detección de abusos antes de que el usuario olvide el cargo, eliminando la pasividad ante los bancos.

---

## 2. Auditoría Manual y Educación Contextual
**Actor:** Ciudadano / Adulto Mayor
**Pre-condición:** El usuario desea revisar un producto específico (ej. un nuevo crédito).

### Flujo Principal:
1. El Usuario selecciona el modo **MANUAL** en el Beeper.
2. El Usuario elige el tipo de producto y el seguro que sospecha es abusivo.
3. El Sistema procesa la solicitud comparando con la base de datos de "ventas atadas".
4. El Sistema entrega un consejo de seguridad y una explicación de la ley (ej. Ley 19.496).
5. El Usuario pulsa "OÍR" para que el Beeper lea el reporte (Accesibilidad).

**Valor Generado:** Democratización del conocimiento financiero y reducción de la brecha digital mediante soporte de voz.

---

## 3. Gestión de Soberanía y Portabilidad
**Actor:** Ciudadano (Dueño de los datos)
**Pre-condición:** El usuario desea controlar su privacidad.

### Flujo Principal:
1. El Usuario pulsa el botón físico de **Escudo (🛡️)** en el Beeper.
2. El Sistema muestra el panel de **MI SOBERANÍA** con los consentimientos activos.
3. El Usuario desactiva o activa permisos de procesamiento (IA, Portabilidad).
4. El Usuario selecciona **"Descargar Mi Pasaporte"**.
5. El Sistema genera un archivo JSON enmascarado con el historial de auditorías.

**Valor Generado:** Transparencia absoluta y cumplimiento de la Ley 19.628, otorgando portabilidad real de los datos.

---

## 4. Empoderamiento Legal (Acción Inmediata)
**Actor:** Ciudadano / Servicio al Cliente (Indirecto)
**Pre-condición:** Se ha detectado un abuso financiero en una auditoría previa.

### Flujo Principal:
1. El Usuario pulsa el botón **"SOLICITAR RENUNCIA"** en la tarjeta de alerta.
2. El Sistema genera una plantilla de carta legal personalizada basada en el tipo de producto.
3. El Usuario revisa la carta generada automáticamente con las leyes correspondientes.
4. El Usuario selecciona **"ENVIAR / COPIAR"** para formalizar el reclamo.
5. El Sistema registra la acción en el log de transparencia.

**Valor Generado:** Reducción de costos legales y eliminación del miedo a reclamar contra grandes instituciones.
